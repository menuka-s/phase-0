This week, we learned a lot about version control and how to use git to accomplish this. Version control is the documentation of changes to a project over time. It basically is a paper trail to show when certain things were done, such as the addition of a feature or the failure of another. Version control allows developers to easily compare new code to old versions at any time.

One system for version control is git. Git offers developers the ability to make "commits" or save checkpoints of their work and document them. It also allows the creation of branches of code from the main master project. This means that a developer can make a copy of the code to edit, while the working master copy stays put. Once the developer feels that their branch is ready, it can easily be merged into the master project.

So git lives on your computer, how do we connect with other developers around the world? Through GitHub! GitHub is really convenient to store code because it allows anyone to make copies or forks. Anyone can edit the code as they see fit and if they want they can make a pull request to contribute their code to the master project. GitHub allows a natural way for teams to know what others have worked on and allows an easy way to review changes that other teammates have made.
